# 学生积分商城 - 后端开发文档

## 1. 项目概述

### 1.1 项目背景
本系统是面向学校学生的积分商城平台，旨在通过积分激励机制，奖励学生的学习表现和成绩，促进学生积极进取。积分兑换全程由教师操作管理。

### 1.2 技术栈
| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | FastAPI | 0.100+ |
| 数据库 | SQLite | 3.x |
| ORM | SQLAlchemy | 2.0+ |
| Excel处理 | openpyxl | 3.1+ |
| 认证 | python-jose | 最新 |
| 密码哈希 | passlib | 1.7+ |
| Python | Python | 3.10+ |

## 4. 业务逻辑

### 4.1 积分兑换流程
1. 教师选择学生和商品
2. 系统检查学生积分是否足够
3. 系统检查商品库存是否充足
4. 扣除学生积分（生成 points_records，type=redeem）
5. 扣除商品库存
6. 生成订单（status=pending）
7. 教师发货后更新订单状态为 completed

### 4.2 抽奖流程
1. 教师选择学生，设置消耗积分
2. 系统检查学生积分是否足够
3. 扣除学生积分（生成 points_records，type=lottery）
4. 根据奖品概率随机抽取奖品
5. 生成抽奖记录（lottery_records）
6. 如果中奖，更新学生积分（加回奖品对应积分或发放实物）

### 4.3 Excel导入流程
1. 教师上传Excel文件
2. 系统解析Excel数据
3. 逐行校验数据（学号、姓名匹配）
4. 返回预览数据，供教师确认
5. 教师确认后批量更新学生积分
6. 生成积分变动记录

### 4.4 积分计算规则
- 学生积分可为负数（扣除时不做下限检查，由业务层判断）
- 积分变动记录中 change_amount 为正数表示增加，为负数表示减少
- 兑换商品和抽奖消耗积分时，检查积分是否充足

## 8. 分页规范

所有列表接口支持分页，参数通过查询字符串传递：
- `page`：页码，从1开始
- `pageSize`：每页数量，默认10

响应数据中包含分页信息：
```json
{
  "code": 200,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```
